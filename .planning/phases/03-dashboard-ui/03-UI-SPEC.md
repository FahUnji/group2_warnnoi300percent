## Component Inventory

All components use CSS Modules (`*.module.css`). No inline styles for new sections — new styles go into `frontend/src/pages/DashboardPage.module.css` (create if not exists).

### 1. ProjectGrid (new section inside DashboardPage)

Displays all synced projects from `GET /api/projects` as a card grid. Clicking a card sets it as active project.

**Layout:**
- CSS grid: `grid-template-columns: repeat(auto-fill, minmax(200px, 1fr))`, gap 16px
- Placed at the top of the dashboard main content area, above summary cards
- Section heading: "Projects" — 20px, weight 700, color `#065b41`, margin-bottom 8px

**States:**

| State | Visual |
|-------|--------|
| Loading (fetching projects) | `<LoadingSpinner size={20} />` + "Loading projects…" caption (14px, color `#6b7280`), centered in grid area |
| Populated | Grid of ProjectCard components |
| Empty (0 projects returned) | Empty state — see Copywriting Contract |
| Error (fetch failed) | Error banner with `role="alert"` — see Copywriting Contract |

**ProjectCard anatomy:**

| Property | Value |
|----------|-------|
| Min-height | 80px |
| Padding | 16px |
| Border-radius | 8px |
| Border (inactive) | `1px solid #c3c6d6` |
| Background (inactive) | `#ffffff` |
| Hover (inactive) | `background: #f3f4f6`, `cursor: pointer` |
| Border (active) | `3px solid #1b4332` |
| Background (active) | `#e7f4f0` |
| Focus-visible | `outline: 2px solid #1b4332; outline-offset: 2px` |
| Layout | `display: flex; flex-direction: column; gap: 4px` |
| Top row | Project avatar (32×32px, background `#e7f4f0`, border-radius 8px, initials in `#1b4332` 14px bold) |
| Name | 14px, weight 700 (active) / 400 (inactive), color `#002d1c` |
| Key badge | 12px, weight 400, color `#434654` |
| Transition | `border-color 0.15s, background 0.15s` |

ARIA: `role="button"` or `<button>` element, `aria-pressed={isActive}`.

### 3. ChartsSection (new section inside DashboardPage)

Two side-by-side pie charts: status breakdown (left) and priority breakdown (right).

**Layout:**
- `display: flex; gap: 24px; flex-wrap: wrap` on the container
- Each chart wrapper: `flex: 1; min-width: 280px`
- Section heading per chart: "Bug Status" (left) and "Bug Priority" (right) — 20px, weight 700, color `#065b41`, margin-bottom 8px
- Chart container padding: 16px, border `1px solid #c3c6d6`, border-radius 8px, background `#ffffff`

**Recharts spec:**

```
<PieChart width={240} height={240}>
  <Pie
    data={distribution}
    cx="50%"
    cy="50%"
    innerRadius={60}
    outerRadius={100}
    paddingAngle={2}
    dataKey="value"
  >
    {distribution.map((entry, idx) => (
      <Cell key={idx} fill={sliceFill(entry.name)} />
    ))}
  </Pie>
  <Tooltip formatter={(value, name) => [value, name]} />
  <Legend verticalAlign="bottom" height={36} />
</PieChart>
```

- Donut style: `innerRadius={60}`, `outerRadius={100}`
- Legend: below the chart (`verticalAlign="bottom"`), default Recharts legend layout
- Tooltip: enabled, shows label + count on hover
- `fill` values: use slice palette tables from Color section above
- Distribution arrays computed on frontend from `bugs` state — no new backend endpoint

**Loading state:** `<LoadingSpinner size={24} />` centered in chart container while `loadingBugs === true`.

**Empty state (0 bugs):** Centered text "No bugs to display" (14px, color `#6b7280`) inside the chart container — no chart rendered.

**No project selected state:** Show "Select a project to view charts" (14px, color `#6b7280`) in each chart container — no chart rendered.

## Interaction Contracts

### Project Card Selection Flow

```
DashboardPage mounts
  → Read sessionStorage('active_project_key')
  → fetch GET /api/projects
  → [loading: spinner in ProjectGrid area]
  → [success] → render ProjectGrid
  → If active_project_key exists → pre-select matching card → fetch GET /api/bugs/{key}
  → [loadingBugs: spinner in SummaryCards + ChartSection, counts show "—"]
  → [bugs loaded] → compute summaries + distributions → render cards + charts

User clicks a different ProjectCard
  → Card enters active state (green border + tint)
  → sessionStorage.setItem('active_project_key', key)
  → fetch GET /api/bugs/{key}
  → [loadingBugs] → counts show "—", charts show spinner
  → [success] → update summaries + distributions → charts re-render
  → [error] → error banner appears below section heading; cards + charts remain in prior state or empty
```

No page navigation on project switch. Everything updates in-place within DashboardPage.

### Sync Flow (unchanged from Phase 2)

```
User clicks Sync Now
  → Button: disabled, opacity 0.7, spinner + "Syncing…"
  → POST /api/sync/{project_key}
  → [success] → button idle; LastSyncedTimestamp updates; fetchBugs re-runs
  → [error] → button idle; error banner appears
```

## Accessibility Baseline

| Requirement | Contract |
|-------------|---------|
| ProjectCard ARIA | Each card is a `<button>` element; `aria-pressed={isActive}` |
| ProjectGrid loading | `<LoadingSpinner>` with `role="status"` + `aria-label="Loading projects"` |
| SummaryCards | Wrap count + label in a `<dl>` or plain `<div>` — no interactive role needed (display-only) |
| Charts | Wrap each `<PieChart>` in `<figure>` with `<figcaption>` matching the chart heading |
| Recharts tooltip | Recharts default tooltip is keyboard-accessible; do not suppress |
| Error banners | `role="alert"` so screen readers announce immediately — carry-forward from Phase 2 |
| Focus visible | All interactive elements: `outline: 2px solid #1b4332; outline-offset: 2px` — do not suppress |
| Spinner ARIA | `role="status"` + `aria-label="Loading"` — already on `LoadingSpinner` component |
