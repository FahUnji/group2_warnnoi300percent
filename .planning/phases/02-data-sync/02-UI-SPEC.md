## Component Inventory

Components that Phase 2 must produce or modify (all CSS Modules, no library):

### 1. ProjectList (new — inside NoProjectPage)

Replaces the existing URL text input and Connect button. Renders the list of Jira projects returned by `GET /api/projects`.

**States:**

| State | Visual |
|-------|--------|
| Loading (fetching project list) | `<LoadingSpinner size={24} />` centered in the list area with "Loading projects…" caption (14px, color `#6b7280`) |
| Populated | Scrollable list (max-height 320px, overflow-y auto) of project rows |
| Empty (API returns 0 projects) | See Copywriting Contract — "No projects found" empty state |
| Error (fetch failed) | See Copywriting Contract — error banner above the list area |
| Project selected | Selected row gets `background: #e7f4f0`, left border `3px solid #1b4332`, `font-weight: 700` on project name |
| Syncing (auto-sync in progress) | Full-width inline status bar replaces list: `<LoadingSpinner size={20} />` + "Syncing {PROJECT_NAME}…" text — 14px, color `#4b5563`, background `#f3f4f6`, border-radius 6px, padding 8px 16px |
| Sync error | Error banner: background `#fef2f2`, border `1px solid #fecaca`, text `#b91c1c`, 13px |
| Sync success | Brief success flash (background `rgba(227,239,234,0.6)`, text `#1b4332`) then immediate redirect — no manual dismiss needed |

**Project row anatomy:**
- Height: 48px minimum
- Padding: 8px 16px
- Layout: flex row, align-items center, gap 12px
- Left: project avatar/icon placeholder — 32x32px, background `#e7f4f0`, border-radius 8px, flex-shrink 0
- Center: project name (14px, 400 weight, `#002d1c`) + project key (12px, 400, `#434654`) stacked vertically
- Right: chevron icon (16x16px, stroke `#c3c6d6`) — hidden on selected row, shown on hover
- Separator: 1px `#c3c6d6` border-bottom on all rows except last
- Hover: background `#f3f4f6`, cursor pointer
- Focus-visible: `outline: 2px solid #1b4332; outline-offset: -2px`

**Section label above list:**
- Text: "SELECT A PROJECT" (see Copywriting Contract)
- Style: 11px, weight 700, color `#065b41`, letter-spacing 0.55px, line-height 16px — matches `.connectLabel` exactly

## Interaction Contracts

### NoProjectPage — Project Selection Flow

```
User lands on NoProjectPage
  → Page mounts → fetch GET /api/projects
  → [loading state: spinner + "Loading projects…"]
  → [success] → render ProjectList rows
  → User clicks a project row
  → Row enters selected state (green left border + bg highlight)
  → Auto-trigger: POST /api/sync/{project_key}
  → List area replaced by syncing status bar: spinner + "Syncing {PROJECT_NAME}…"
  → [sync success] → brief success flash (500ms) → navigate to /dashboard
  → [sync error] → error banner appears above list; list re-renders; user may pick again
```

No separate "Connect" or "Confirm" button — selection is the trigger. This is consistent with D-05 in CONTEXT.md.

### DashboardPage — Manual Sync Flow

```
User is on DashboardPage
  → Sync Now button visible in header, idle state
  → User clicks Sync Now
  → Button enters loading state (disabled, spinner, "Syncing…" label)
  → POST /api/sync/{project_key} fires
  → [success] → button returns to idle; LastSyncedTimestamp updates to "Last synced: just now"
  → [error] → button returns to idle; error banner appears below the header area
```

## Accessibility Baseline

| Requirement | Contract |
|-------------|---------|
| Spinner ARIA | `role="status"` + `aria-label="Loading"` — already on `LoadingSpinner` component |
| Project list ARIA | `role="list"` on the list container; `role="listitem"` or `<button>` on each row |
| Selected project | `aria-pressed="true"` on the selected row button |
| Sync button loading | `aria-busy="true"` + `aria-disabled="true"` during sync |
| Error banners | `role="alert"` so screen readers announce errors immediately |
| Focus visible | All interactive elements must show `outline: 2px solid #1b4332` on focus-visible — do not suppress outlines |
