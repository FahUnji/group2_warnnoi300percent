<decisions_made>

- **chartBody flex**: Changed `grid-template-columns: 1fr auto 1fr` to `display: flex; justify-content: center` — fixed grid with 240px SVG broke at ~380px card width (1000px viewport with sidebar visible)
- **chartsRow breakpoint 1100px**: Was 960px — at 960-1100px viewport, priority card was only 380-480px wide when side-by-side with 340px status card
- **statusChartBody row at ≤960px**: When chartsRow collapses to 1fr, status card gets full width — horizontal layout (donut left, legend right) uses horizontal space
- **severityCards flex-wrap**: 4 items × ~86px + 3 × 48px = 408px, overflows 327px mobile content area without wrapping
- **sprintName ellipsis**: `white-space: nowrap` without overflow control caused row overflow on long sprint names; added `min-width: 0` on sprintMeta parent
- **Navbar logo left-aligned**: User tested centered, preferred left next to hamburger. navLeft is always 36px (empty div on Dashboard, hamburger button on sidebar pages) → logo x-position identical on all pages
- **Removed pointer-events: none from navCenter**: Only needed when navCenter was absolutely positioned and could intercept clicks; removed with centering
</decisions_made>
