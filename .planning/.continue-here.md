<decisions_made>

- **Cookie-based sessions** (not JWT in localStorage) ‚Äî user chose this. HttpOnly blocks XSS, SameSite=Lax blocks CSRF, browser sends automatically.
- **Wipe DB** ‚Äî user chose this. Old schema had no `user_id`; new schema requires `user_id NOT NULL` FK. Migration not feasible.
- **7-day session TTL** ‚Äî `sessions.expires_at` checked against `datetime('now')` per request. Instant revocation possible.
- **COOKIE_SECURE env-driven** ‚Äî defaults `false` for dev (HTTP localhost), set `COOKIE_SECURE=true` for production (HTTPS).
- **OAuth token kept on logout** ‚Äî only session deleted. Re-login creates new session without OAuth dance. Token stays encrypted per `user_id`.
- **jira_config table kept unchanged** ‚Äî API token auth endpoints (`/api/jira/connect`, `/api/jira/status`, `/api/jira/me`) are dead code in current UI (ConnectionPage uses OAuth only). Protected by `Depends(get_current_user)` but `jira_config` remains global. Can be removed later.
</decisions_made>
