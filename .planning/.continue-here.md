## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| database.py create-only init_db | init_db() with only CREATE TABLE IF NOT EXISTS silently fails on schema changes — existing tables never updated | blocking | Always add DROP+recreate migration when schema changes; check `sqlite_master` for column presence |
| Redirect to undefined route | /no-project not in React Router — *-catch-all fires immediately | blocking | Only redirect to routes defined in App.jsx (/, /dashboard, /bug-report, /sprint) |
| jira_session cookie (old) | Previous sessions may have old jira_session cookie — new auth uses session_id | advisory | Always clear cookies before testing after auth changes |
| git add -A or staging .planning/*.md | CRLF noise, unrelated files | advisory | Explicit file paths in git add |
| Edit .py without restarting container | uvicorn --reload blind to NTFS changes | advisory | Always docker restart after .py edit via powershell.exe |
