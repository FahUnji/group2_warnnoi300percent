## Component Inventory

### Phase 1 Components (build these as React components)

#### 1. `ConnectionPage` — full-page layout

Structure: two-column hero grid (7fr / 5fr) — left hero copy + right login card.

| Sub-element | CSS class equivalent | Notes |
|-------------|----------------------|-------|
| Page wrapper | `.page-wrapper` | flexbox centering, padding 80px 64px |
| Hero grid | `.hero-grid` | `grid-template-columns: 7fr 5fr; gap: 24px; max-width: 1200px` |
| Hero left | `.hero-left` | Hidden on mobile (<900px) |
| Login card | `.login-card` | `border-radius: 8px; padding: 48px; box-shadow: 0 40px 80px -20px rgba(0,45,28,0.08)` |
| Footer | `footer` | `background: #f8f9ff; border-top: 1px solid #c3c6d6; padding: 16px 24px` |

**NOTE:** Replace the original "Continue with Atlassian" OAuth button with a 3-field credential form (D-01, D-09).

#### 2. `ConnectionForm` — replaces the CTA button in the card

Fields (exactly 3, per D-01):
- `base_url` — text input, label "Jira Base URL", placeholder `https://yourcompany.atlassian.net`
- `email` — email input, label "Email Address", placeholder `you@company.com`
- `api_token` — password input, label "API Token", placeholder `••••••••••••`

Submit button: full-width, matches `.cta-btn` style — `background: #1b4332; border-radius: 12px; padding: 16px 24px; font-size: 16px; font-weight: 700`.

Input field style (infer from card style, no explicit input CSS in references):
- `border: 1px solid rgba(193,200,194,0.84)`
- `border-radius: 8px`
- `padding: 12px 16px`
- `font-size: 16px; font-family: Inter`
- Focus: `border-color: #1b4332; outline: none; box-shadow: 0 0 0 3px rgba(27,67,50,0.12)`

#### 3. `StatusBanner` — inline status inside the card

Used for loading, success flash, and error states. Three variants:

| Variant | Background | Text color | Usage |
|---------|------------|------------|-------|
| `loading` | `#f3f4f6` | `#4b5563` | During connection test (spinner + "Connecting to Jira...") |
| `success` | `rgba(227,239,234,0.6)` | `#1b4332; font-weight: 700` | Brief success flash before redirect |
| `error` | `#fef2f2` | `#b91c1c` | Inline error after failed connection attempt |

Dimensions: `padding: 8px 16px; border-radius: 8px; font-size: 14px; gap: 8px` (rounded from reference 10px/14px to nearest 4-multiples for grid alignment).

**Source:** UI/css/login.css `.status-banner`, `.status-banner.loading`, `.status-banner.success`, `.status-banner.error`

#### 4. `SuccessModal` — overlaid on page after successful connection

Modal card: `max-width: 380px; border-radius: 12px; padding: 36px 32px 32px; gap: 10px`.

Animation: `slide-up` (translateY 16px → 0, 0.25s) on mount; `pop-in` (scale 0.5 → 1, 0.35s) on icon; `draw-check` (stroke-dashoffset 30 → 0, 0.3s at 0.3s delay) on checkmark path.

Icon: 64×64px mint square (`#e7f4f0; border-radius: 12px`) containing 40×40px dark green circle (`#1b4332`) with animated SVG checkmark.

Behaviour (D-10): shows briefly (2–3s), then auto-redirects to `/dashboard`. "Go to Dashboard" button allows immediate navigation.

Overlay: `background: rgba(55,65,81,0.75); position: fixed; inset: 0`.

#### 5. `LoadingSpinner` — discretion item

During the connection test API call, show inside the `StatusBanner loading` variant. Use a simple CSS animation (rotate) — 20×20px, `border: 2px solid #e5e7eb; border-top-color: #1b4332; border-radius: 50%`. Duration: 0.6s linear infinite.

## Interaction States

### Connection Form States

| State | Trigger | Visual |
|-------|---------|--------|
| Idle | Page load (no saved credentials) | Form fields empty, submit button enabled |
| Submitting | User clicks "Connect to Jira" | Button disabled (opacity 0.6, cursor not-allowed); `StatusBanner loading` appears above button with spinner |
| Success | API returns `{"ok": true}` | `StatusBanner success` flashes; `SuccessModal` overlays; auto-redirect after 2s |
| Error | API returns `{"ok": false}` | `StatusBanner error` with error-specific message inline; `api_token` field cleared (D-11); `base_url` and `email` preserved |
| Auto-verify loading | App load with saved credentials | Full-page loading state (spinner center-screen); redirect to dashboard if valid |

### Error Message Copy by Error Code (D-12)

| Error Code | `StatusBanner` copy |
|------------|---------------------|
| `invalid_credentials` | "Invalid email or API token. Please check your credentials and try again." |
| `unreachable_host` | "Cannot reach Jira at that URL. Check the base URL and your network connection." |
| `timeout` | "Connection timed out. Jira did not respond in time — try again in a moment." |

## Implementation Notes for Executor

1. **No OAuth button:** Replace `.cta-btn` ("Continue with Atlassian") with the 3-field `ConnectionForm` component. Preserve all card chrome (icon, title, subtitle, security badge, divider, footer).

2. **CSS approach:** CSS Modules (`.module.css` per component). Port class names verbatim from reference CSS where possible. No Tailwind, no CSS-in-JS.

3. **Form file structure suggested:**
   - `frontend/src/pages/ConnectionPage.tsx`
   - `frontend/src/components/ConnectionForm/ConnectionForm.tsx` + `ConnectionForm.module.css`
   - `frontend/src/components/StatusBanner/StatusBanner.tsx` + `StatusBanner.module.css`
   - `frontend/src/components/SuccessModal/SuccessModal.tsx` + `SuccessModal.module.css`
   - `frontend/src/components/LoadingSpinner/LoadingSpinner.tsx` + `LoadingSpinner.module.css`

4. **Auto-verify on load (D-14):** On app boot, React calls `GET /api/jira/verify`. If `{"ok": true}` → navigate to `/dashboard`. If `{"ok": false}` → render `ConnectionPage` with optional inline error banner. Show full-screen spinner while verifying.

5. **api_token field clear on error (D-11):** After a failed connection attempt, call `setValue('api_token', '')` (React state reset). Do NOT clear `base_url` or `email`.

6. **Success flash timing (D-10):** Show `SuccessModal` immediately on success response. Auto-redirect after 2000ms via `setTimeout(() => navigate('/dashboard'), 2000)`. "Go to Dashboard" button navigates immediately.

7. **Google Fonts import:** Load in `index.html` or global CSS: `Inter:wght@400;700`, `JetBrains+Mono:wght@400`, `Hanken+Grotesk`, `Inder`.
