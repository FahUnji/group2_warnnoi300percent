**`migrations/001_create_jira_config.sql`** — exact content:
```sql
-- Migration 001: Create jira_config table
-- Run once against jira_db before starting the backend.
-- This is a single-row config table. id=1 is always the active config.
-- ON DUPLICATE KEY UPDATE means re-configuring replaces the existing row.

CREATE DATABASE IF NOT EXISTS jira_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE jira_db;

CREATE TABLE IF NOT EXISTS jira_config (
  id                    INT AUTO_INCREMENT PRIMARY KEY,
  base_url              VARCHAR(500) NOT NULL,
  email                 VARCHAR(255) NOT NULL,
  api_token_encrypted   TEXT         NOT NULL,
  created_at            TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
  updated_at            TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
                          ON UPDATE CURRENT_TIMESTAMP
);
```

**`backend/models/jira_config.py`** — exact content:
```python
"""
jira_config model — single-row table storing Jira connection credentials.
API token is stored encrypted (Fernet). Use upsert_config to save,
load_config to retrieve (returns decrypted token via the service layer).
"""
from backend.database import get_db


def upsert_config(base_url: str, email: str, api_token_encrypted: str) -> None:
    """Insert or replace the single jira_config row."""
    conn = get_db()
    cur = conn.cursor()
    cur.execute(
        """
        INSERT INTO jira_config (base_url, email, api_token_encrypted)
        VALUES (%s, %s, %s)
        ON DUPLICATE KEY UPDATE
            base_url            = VALUES(base_url),
            email               = VALUES(email),
            api_token_encrypted = VALUES(api_token_encrypted),
            updated_at          = CURRENT_TIMESTAMP
        """,
        (base_url, email, api_token_encrypted),
    )
    conn.commit()
    cur.close()
    conn.close()


def load_config() -> dict | None:
    """Return the saved config row as a dict, or None if not yet configured.
    NOTE: api_token_encrypted is returned as-is — decrypt in the service layer.
    """
    conn = get_db()
    cur = conn.cursor(dictionary=True)
    cur.execute("SELECT * FROM jira_config ORDER BY id DESC LIMIT 1")
    row = cur.fetchone()
    cur.close()
    conn.close()
    return row
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser → FastAPI | HTTP POST with Jira credentials (base_url, email, api_token) — untrusted input |
| FastAPI → MySQL | Write of encrypted token — must use parameterized queries only |
| FastAPI → .env file | FERNET_KEY sourced from process env — must never be hardcoded or logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `backend/main.py` CORS | mitigate | `allow_origins=["http://localhost:3000"]` only — no wildcard. Blocks cross-origin credential theft from other local ports. |
| T-01-02 | Information Disclosure | `backend/models/jira_config.py` | mitigate | `api_token_encrypted TEXT` column — plain token never written to DB. Fernet encrypt before every INSERT. |
| T-01-03 | Information Disclosure | `backend/database.py` + `.env` | mitigate | All DB credentials via `os.environ.get()` — no hardcoded passwords. `.env.example` in repo; `.env` in `.gitignore`. |
| T-01-04 | Information Disclosure | `FERNET_KEY` env var | mitigate | Key loaded from `os.environ["FERNET_KEY"]` — raises `KeyError` if unset rather than using a silent default. Key never logged, never returned in API responses. |
| T-01-05 | Tampering | `migrations/001_create_jira_config.sql` | accept | Migration runs locally by developer. No automated migration runner in Phase 1 — low risk; no remote execution path. |
| T-01-06 | Denial of Service | FastAPI startup crash (missing env vars) | accept | If `DB_*` vars missing, mysql.connector raises on first request (not startup). Acceptable for dev environment — document in .env.example. |
| T-01-07 | Elevation of Privilege | SQL injection via jira_config upsert | mitigate | All values passed as `%s` parameterized placeholders in `cur.execute()` — no string interpolation in SQL. |
</threat_model>
